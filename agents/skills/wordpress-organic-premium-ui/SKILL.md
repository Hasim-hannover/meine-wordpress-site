# WordPress Organic Premium UI

Use this skill when the site should feel expensive and clear without becoming visually dense or overly polished.

## Primary Scope

- `blocksy-child/assets/css/design-system.css`
- `blocksy-child/assets/css/homepage.css`
- `blocksy-child/assets/css/wgos.css`
- `blocksy-child/assets/css/agentur.css`
- `blocksy-child/assets/css/footer-cta.css`
- relevant templates or partials that shape trust and CTA surfaces

## Design Direction

- Keep the dark foundation, but move from black-glass heaviness to softer graphite surfaces.
- Use restrained warm accents, not loud orange everywhere.
- Prefer fewer, calmer surfaces with better spacing over many nested cards.
- Introduce subtle organic gradients or haze, not decorative noise.
- Make trust modules feel editorial and composed, not gadget-like.

## Refine in This Order

1. `Tokens`
   Tune palette, border strength, shadows, radii, and button behavior first.
2. `High-visibility components`
   Hero, proof blocks, CTA boxes, trust modules, cards, and buttons.
3. `Page wrappers`
   Homepage, WGOS, and service-page backgrounds should feel related.
4. `Exception cleanup`
   Remove mismatched accents, emoji-like trust markers, and over-animated elements when they cheapen the page.

## Guardrails

- Do not flatten the design into generic SaaS minimalism.
- Do not add purple gradients, default UI kits, or trendy effect stacks.
- Avoid thick borders and overly bright hover states.
- Premium here means calm, specific, and breathable.
- If a page is editor-driven, keep repo changes structural and list any manual editor cleanup separately.

## Expected Output

When you finish, separate the result into:

- `Token changes`
- `Component changes`
- `Page-specific changes`
- `Manual WP follow-up`
