# Components

Use this file when building or revising hero sections, proof surfaces, offers, CTA blocks, or page architecture.

## Hero

Job:

- state the value proposition fast
- orient the visitor
- trigger the first action

Must include:

- one `H1`
- one supporting paragraph
- one primary CTA
- one trust signal above the fold
- one proof or diagnostic surface nearby

Patterns:

- On desktop, prefer an asymmetric layout: copy on one side, proof or diagnostic panel on the other.
- Centered hero text is acceptable only when the line length stays tight and the screen still has a strong focal point.
- Keep the headline visually dominant and the CTA surrounded by breathing room.

## Proof Bar

Place proof within one scroll:

- metrics with context
- client logos with treatment, not a flat strip
- case teaser with one concrete outcome
- proof bullets tied to the offer, not vanity metrics

Avoid weak proof:

- empty "trusted by" bars
- unlabeled numbers
- testimonial carousels

## Mechanism / Process

Explain how the system works in three steps when possible:

1. diagnosis
2. blueprint or prioritization
3. implementation or optimization

Make the middle step visually strongest if it is the economic upgrade path.

## Offer Section

The offer section should clarify what the buyer gets, not dump services.

Prefer:

- tiered progression
- outcome framing
- scope boundaries
- explicit "not for you" statements when they increase qualification

## FAQ / Objection Handling

Use FAQ to remove resistance, not to restate features.

Good topics:

- fit
- timeline
- scope
- dependency on ads or tools
- what happens after the first engagement

## Final CTA

Repeat the primary action with more confidence and less complexity.

Good supporting copy:

- short recap of the payoff
- low-friction framing
- one small reassurance line
